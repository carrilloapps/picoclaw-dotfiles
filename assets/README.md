# Assets

Photos and screenshots of the actual PicoClaw deployment for documentation and GitHub social preview.

## Files

| File | Description | Used in |
| ---- | ----------- | ------- |
| `neofetch.jpg` | Terminal output showing device specs (1897×896) | README.md gallery, GitHub social preview |
| `photo-back.jpg` | Back of the Xiaomi Redmi Note 10 Pro (1200×1600) | README.md gallery |
| `photo-front.jpg` | Front of the device running Termux (1200×1600) | README.md gallery |
| `screenshot-termux.png` | Termux terminal session on the device (1080×2400) | README.md gallery |

## Adding New Assets

When adding screenshots or photos:

1. Compress images (keep under 500 KB for fast loading)
2. Use descriptive filenames: `screenshot-<what>.png`, `photo-<what>.jpg`
3. Reference in documentation with relative paths: `![Alt](../assets/filename.jpg)`
4. Prefer `.jpg` for photos, `.png` for screenshots with text
